# 隐私保险箱 App -- UI 设计规范

**Status**: Draft | **Last Updated**: 2026-03-31 | **Version**: 1.0

---

## 1. 设计理念与风格定位

### 1.1 核心设计关键词

**安全 -- 私密 -- 简洁 -- 现代 -- 信任**

隐私保险箱的视觉语言建立在一个核心命题上：**让用户在打开 App 的第一秒就感受到"这里是安全的"**。不靠花哨的动效，不靠拟物的金属锁，而是用克制、精确、有呼吸感的界面传递专业与可靠。

### 1.2 风格定义

- **暗色基调为主**：深色背景传递私密感和安全感，同时减少 OLED 屏幕功耗、降低夜间使用刺眼程度
- **低饱和度色彩**：避免高饱和色带来的廉价感，主色调采用沉稳的蓝色系
- **大量留白**：内容区留出充足呼吸空间，不堆砌元素
- **圆角柔和**：卡片和按钮使用中等圆角（8-12px），传递友好但不失专业
- **层级分明**：通过背景色深浅区分层级，而非依赖阴影和描边

### 1.3 与竞品的视觉差异化

| 维度 | 竞品现状 | 我们的策略 |
|------|---------|-----------|
| 配色 | 多数使用亮蓝/亮绿，部分使用金色"保险箱"隐喻 | 深色主基调 + 低饱和蓝色强调，传递专业而非花哨 |
| 图标 | 锁、盾牌等安全隐喻图标，风格混杂 | 统一的线性图标，极简几何风格，不用锁/盾等直白隐喻 |
| 布局 | 信息密度高，操作按钮杂乱 | 大量留白，操作收敛到统一入口（FAB + 更多菜单） |
| 字体 | 系统默认，层级不清晰 | 严格的字号阶梯，中文用系统默认（Noto Sans CJK SC），英文/数字用等宽字体突出数据感 |
| 伪装 | 计算器伪装 UI 粗糙 | 计算器界面做到和系统计算器几乎一致，不露破绽 |

### 1.4 设计灵感方向

- **Signal / Telegram**：加密通信 App 的极简界面风格，传递安全感
- **Apple Files / Google Files**：文件管理的清晰信息层级和网格布局
- **1Password / Bitwarden**：安全工具类 App 的深色基调和信任感传递
- **MIUI 计算器**：计算器伪装的视觉参考标杆，追求与系统原生一致

### 2.5 色彩应用规则

1. **所有颜色必须从 Token 取值**，代码中禁止写死 HEX 值
2. 文字颜色只使用 `neutral` 系列，不允许使用主色或语义色作为正文颜色
3. `error.main` 仅用于错误提示和破坏性操作（删除、销毁），不用于一般的红色装饰
4. 交互元素的对比度必须满足 WCAG AA（文字 4.5:1，大文字 3:1）
5. 图标颜色跟随文字层级，不单独定义颜色

## 3. 字体排版

### 3.1 字体选择

| 用途 | 字体 | 说明 |
|------|------|------|
| 中文 | `Noto Sans CJK SC` (系统默认) | 系统字体，无需打包，渲染一致 |
| 英文/数字 | `Roboto` | Flutter 默认，与中文字体搭配协调 |
| 等宽数字 | `Roboto Mono` | PIN 输入、计算器显示屏、文件大小等数据场景 |
| 计算器显示屏 | `Roboto Mono` | 等宽确保数字对齐，模拟真实计算器 |

### 3.2 字号层级体系（Type Scale）

基于 4px 网格，所有字号均为 4 的倍数或半倍：

| Token | 字号 | 字重 | 行高 | 用途 |
|-------|------|------|------|------|
| `display` | 48px | 700 (Bold) | 1.2 (58px) | 计算器显示屏结果数字 |
| `h1` | 28px | 700 (Bold) | 1.3 (36px) | 页面大标题（极少使用） |
| `h2` | 22px | 600 (SemiBold) | 1.3 (28px) | 页面标题（设置页标题、导入页标题） |
| `h3` | 18px | 600 (SemiBold) | 1.4 (24px) | 分组标题、对话框标题 |
| `body.lg` | 16px | 400 (Regular) | 1.6 (26px) | 主要正文、列表项主文字 |
| `body.md` | 14px | 400 (Regular) | 1.5 (22px) | 次要正文、说明文字、输入框文字 |
| `body.sm` | 12px | 400 (Regular) | 1.5 (18px) | 辅助文字、时间戳、文件大小、标签 |
| `caption` | 10px | 400 (Regular) | 1.4 (14px) | 角标、极小提示（尽量少用） |
| `button.lg` | 16px | 600 (SemiBold) | 1.0 (16px) | 主按钮文字 |
| `button.md` | 14px | 500 (Medium) | 1.0 (14px) | 次级按钮、Tab 文字 |
| `button.sm` | 12px | 500 (Medium) | 1.0 (12px) | 小按钮、操作链接 |

### 3.3 字重使用规范

| 字重 | 数值 | 使用场景 |
|------|------|---------|
| Regular | 400 | 正文、说明文字、输入框内文字 |
| Medium | 500 | 次级按钮、Tab 标签、导航栏标题 |
| SemiBold | 600 | 页面标题、分组标题、对话框标题 |
| Bold | 700 | 计算器数字、核心数据展示、品牌名称 |

**禁止使用**：Light (300)、Thin (100) -- 在移动端小屏幕上可读性差。

### 3.4 字间距

| 场景 | 字间距（letter-spacing） |
|------|------------------------|
| 标题 | 0px（默认） |
| 正文 | 0px（默认） |
| 按钮 | 0.5px（略微加宽） |
| 计算器数字 | 1px（等宽加宽，增强数据感） |
| 全大写英文 | 1.5px（如 "AES-256"） |

## 4. 图标系统

### 4.1 图标风格定义

- **风格**：线性图标（Outlined），描边粗细 1.5px
- **风格一致性**：所有图标统一使用 Material Symbols Outlined 图标库
- **圆角**：图标内部转角统一使用 2px 圆角
- **填充模式**：仅在"选中/激活"状态使用填充（Filled）变体，形成明确的状态区分

### 4.4 App 图标设计

#### 计算器伪装图标（桌面展示）

- **外形**：标准 Android 自适应图标（圆角方形）
- **背景色**：`#2D2D2D`（深灰，接近系统计算器）
- **前景**：白色简约计算器图案（显示屏 + 按键网格的抽象图形）
- **风格**：与各大厂商系统计算器图标风格保持一致，不突兀

#### 真实品牌图标（应用内/关于页展示）

- **外形**：圆角方形
- **背景色**：渐变 `#3D64A8` -> `#233D7A`
- **前景**：白色抽象几何图案 -- 一个由线条构成的六边形护盾轮廓，内部一个简约的锁孔形状
- **风格**：极简、几何、现代，不使用拟物效果

```
+----------------------------------------------+  <- 状态栏（系统默认）
```

**设计要点**：
- 显示屏区域占页面上半部分约 35%，下半部分为键盘

- 功能键（C、+/-、%）背景 `neutral.100`（浅色）或 `neutral.700`（深色）
- 运算符（/、x、-、+、=）背景 `primary.500`，文字白色
- 按键文字：数字 24px Regular，运算符 28px Medium
- 按键按下态：背景色加深 15%，无涟漪效果（接近系统计算器行为）
- 当前被选中的运算符高亮显示（背景变为 `primary.400`）
- 整体风格尽量模仿当前主流 Android 系统计算器（MIUI / 原生）

## 8. 动效与微交互

### 8.1 转场动画

| 场景 | 动画类型 | 时长 | 缓动曲线 | 描述 |
|------|---------|------|---------|------|
| 计算器 -> 主页 | 形变过渡 | 400ms | `easeInOutCubic` | 计算器显示屏向上收缩，键盘淡出，主页内容从中心展开 |
| 页面前进 | 共享元素 + 淡入 | 300ms | `easeOutCubic` | 新页面从右侧滑入 30%，同时淡入；当有共享元素（如缩略图 -> 全屏图片）时使用 Hero 动画 |
| 页面返回 | 淡出 + 滑出 | 250ms | `easeInCubic` | 当前页面向右滑出 30%，同时淡出 |
| Tab 切换 | 交叉淡入淡出 | 200ms | `easeInOut` | 内容区域淡出淡入，Tab 指示器平滑滑动 |
| 对话框弹出 | 缩放 + 淡入 | 200ms | `easeOutBack` | 从 0.9 缩放到 1.0，同时淡入 |
| 底部弹出表 | 滑入 | 250ms | `easeOutCubic` | 从底部滑入到目标位置 |

### 8.2 按钮点击反馈

| 按钮类型 | 反馈方式 |
|---------|---------|
| 主按钮 / 次级按钮 | 按下时缩放 0.98（100ms），释放后弹回 1.0（150ms） |
| FAB | 按下时缩放 0.95 + 阴影缩小（100ms） |
| 图标按钮 | Material InkWell 涟漪效果，半径 20dp |
| 计算器按键 | 背景色加深 15%（50ms），无涟漪，模拟物理按键感 |
| 危险按钮 | 同主按钮，但短震动触觉反馈 |

### 8.3 加密进度的视觉表现

```
进度弹窗：
+--------------------------------------+
|                                      |
|          +------+                    |
|         /  78%  \                    |  <- 圆环进度动画
|        |        |                    |      平滑填充
|         \      /                     |      颜色: primary.500
|          +------+                    |
|                                      |
|     正在加密 15/20 ...                |  <- 文件计数器
|     当前: IMG_0042.jpg               |  <- 当前文件名
|                                      |
|     [最小化]                          |  <- 文字按钮
|                                      |
+--------------------------------------+
```

- 圆环进度采用平滑动画，不跳跃
- 文件计数器每完成一个文件更新
- 最小化后在应用内顶部显示一条细进度条（高 3px，`primary.500`）
- 全部完成后弹出 Toast "成功导入 20 个文件"，圆环变为绿色勾号动画

### 8.4 解锁成功的视觉反馈

**时间线（总计 600ms）**：

```
T=0ms      PIN 最后一位输入 -> 圆点全部变为 primary.500 (实心)
T=100ms    圆点变为 success.main，短震动反馈
T=100ms    锁图标开始开锁动画：
           - 锁扣向上旋转 45 度 (200ms)
           - 锁体淡出 (100ms)
T=400ms    整个解锁页内容向上移动 20px 并淡出 (200ms)
T=500ms    主页内容从透明度 0 淡入 (200ms)，同时从下方移入 10px
T=700ms    动画完成，主页完全可交互
```

### 8.5 文件删除动画

```
T=0ms      被删文件卡片开始缩小 (scale 1.0 -> 0.0, 200ms)
T=0ms      同时透明度 1.0 -> 0.0
T=200ms    其余卡片平滑重排填补空位 (200ms, easeInOut)
T=400ms    Snackbar 从底部滑入 "已删除 1 个文件" [撤销]
```

### 8.6 多选模式激活

```
T=0ms      长按文件触发：
           - 手机震动反馈（中等强度，50ms）
           - 被长按的文件卡片轻微缩放 0.95 (100ms)
T=100ms    所有文件卡片同时轻微缩放到 0.97 (150ms, easeOut)
T=100ms    每个卡片右上角勾选框从 scale 0 弹出 (200ms, easeOutBack)
T=150ms    导航栏背景色变为 primary.500 (200ms 过渡)
T=200ms    底部批量操作栏从底部滑入 (200ms)
T=250ms    被长按的文件自动选中（勾选框填充 primary.500）
```

### 9.2 深色模式阴影处理

深色模式下**不使用阴影**（阴影在深色背景上不可见且会造成脏灰感）。层级关系通过背景色的深浅变化来表达：

```
层级关系（深色模式）:
底层 (页面背景):    neutral.900 (#111827)
抬升层 (卡片):      neutral.800 (#1F2937)
浮层 (对话框):      neutral.800 (#1F2937) + 1px border neutral.700
最上层 (弹出菜单):   neutral.700 (#374151)
```

### 9.3 图片/视频预览在深色模式下的处理

| 元素 | 处理 |
|------|------|
| 全屏预览背景 | 始终使用 `neutral.950` (#0A0E17)，无论浅色/深色模式 |
| 缩略图网格 | 间距处显示页面背景色 |
| 视频播放器控件 | 始终使用白色文字/图标 + 半透明黑色遮罩，无论模式 |
| 图片预览操作栏 | 同视频播放器，始终使用暗色遮罩模式 |

### 9.4 语义色在深色模式下的调整

深色模式下，语义色的"浅底色"变体需要降低明度、提高透明度：

| Token | 浅色模式 | 深色模式 |

## 10. 适配规范

### 10.1 屏幕尺寸适配策略

| 屏幕类别 | 宽度范围 | 策略 |
|---------|---------|------|
| 小屏 | < 360dp | 缩略图网格 2 列，文字不缩小（最小 12px），间距适当压缩 |
| 标准 | 360-411dp | **主要设计尺寸**，3 列缩略图，标准间距 |
| 大屏 | 412-599dp | 3-4 列缩略图，间距保持不变 |
| 平板 | >= 600dp | 4-5 列缩略图，页面最大宽度 600dp 居中 |

### 10.2 关键断点

| 断点 | 值 | 影响 |
|------|-----|------|
| 文件夹网格列数 | 始终 2 列 | 不随屏幕变化 |
| 缩略图默认列数 | < 360dp: 2 列 / >= 360dp: 3 列 | 可通过捏合手势 2-5 列切换 |
| 页面最大宽度 | 600dp | 大屏和平板上内容居中，不拉伸 |
| 计算器按键 | 按键宽度 = 屏幕宽度 / 4 | 等分 4 列自适应 |
| PIN 键盘 | 按键固定 64px 可见区域 | 间距自适应 |

### 10.3 刘海屏 / 挖孔屏适配

| 适配项 | 处理方式 |
|--------|---------|
| 状态栏高度 | 使用 `MediaQuery.of(context).padding.top`，不硬编码 |
| 导航栏 | 背景色延伸到状态栏区域，内容区域从安全区域开始 |
| 刘海区域 | 导航栏标题和图标不与刘海区域重叠 |
| 挖孔屏 | 挖孔通常在状态栏区域内，无需额外处理 |
| 底部手势条 | 使用 `MediaQuery.of(context).padding.bottom`，FAB 和底部工具栏位于手势条上方 |

### 10.4 Flutter 实现要点

```
// 安全区域使用
SafeArea(
  child: content,
)

// 或更精细的控制
Padding(
  padding: EdgeInsets.only(
    top: MediaQuery.of(context).padding.top,
    bottom: MediaQuery.of(context).padding.bottom,
  ),
  child: content,
)
```

### 10.5 折叠屏考虑（可选）

由于 MVP 阶段不需要特别支持折叠屏，但为未来预留考虑：

| 场景 | 策略 |
|------|------|
| 折叠状态 | 与普通手机一致 |
| 展开状态（大屏） | 使用平板适配策略，内容最大宽度 600dp 居中 |
| 半折叠状态 | 不特殊处理，使用普通手机布局 |

### 10.6 字体大小适配

| 策略 | 说明 |
|------|------|
| 系统字体缩放 | 尊重系统字体大小设置（`MediaQuery.textScaleFactor`） |
| 最大缩放限制 | 最大 1.3x，防止布局溢出 |
| 关键区域 | PIN 键盘、计算器键盘的文字不跟随系统缩放，保持固定大小 |
| 测试要求 | 在 0.85x 和 1.3x 缩放下均能正常显示 |
